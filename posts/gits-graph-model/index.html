Git’s Graph Model

Directed Acyclic Graph (DAG)

Graph

  • A way to model connected things
  • Contain nodes connected by edge

Direcrted Graph

  • Nodes are connected in a certain director

Arrow direction

  • Direction depends on how you define the relationship

Acyclic

  • Acyclic mean “no cycles” or “non-circular”

DAG

  • Contains nodes connected with arrows and has no cycles

Git DAG

  • Git models the relationship of commits with a DAG
  • The arrows point at a commit’s parent(s)

Branch

  • Occurs if a commit has more than one child

Merge

  • A merge occurs when a commit has more than one parent

Git IDs

Git Objects

  1. Commit - A small text file
  2. Annotated tages - A permanent reference to a commit
  3. Tree - Directories and filenames in the project
  4. Blob - The content of a file in the project

Git IDs

  • The name of a Git object
  • 40-character hexadecimal sting
  • Also known as object ID, SHA-1, hash and checksum

Secure Hash Algorithm 1 (SHA-1)

  • Git IDs are SHA-1 values
  • Unique for a given piece of content (statistically speaking)
SHA-1 is designed to avalanche

Creating a SHA-1 for file contents

Use git hush-object <file> to creat an SHA-1 for any content

Shortened Git IDs

  • Four or more characters of the beginning of a Git ID
