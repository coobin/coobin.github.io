Review

  • A centralized workflow involves working on a single branch
  • In a feature branch workflow, work of the project is done on feature/topic branches and then merged into longer running branches
  • In a forking workflow, work is added upstream using pull request from the forked repository
  • Gitflow workflows enable a continuous train of project release using multiple types of branches
