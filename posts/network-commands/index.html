Network commands

Clone - copies a remote repository Fetch - Retrieves new objects and references from the remote repositiry Pull - Fetches and merges commits locally Push - Adds new objects and references to the remote repository

Fetch

git fetch <repository>

  • Retrieves new objects and references from another repository
  • Tracking branches are updated

Fetching the latest commits

After git fetch git status will inform you that your current branch is behind the tracking branch

Pull

git pull [\<repository>] [\<branch>] Combines git fetchandgit merge FETCH_HEAD`

  • If objects are fetched, the tracking branch is merged into the current local branch
  • This is similar to a topic branch merging into a base branch

git pull merging options

–ff (default) - fast-forward if possibe, otherwise perform a merge commit –no-ff - always include amerge commit –ff-only - cancel instead of doing a merge commit –rebase [–preserve-merges] - discussed later …….
